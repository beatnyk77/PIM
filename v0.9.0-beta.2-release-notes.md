# PIM v0.9.0-beta.2 Release Notes

We are thrilled to announce the release of **PIM v0.9.0-beta.2**, which finalizes the security, trust-boundary enforcement, and local auditing layers of our decentralized **MLS-Aligned Secure Group Messaging** system.

This release represents a critical stabilization checkpoint, bringing cryptographic invite protections, localized security audit ledgering, and E2EE media attachment capabilities to PIM.

---

## 🚀 Key Improvements & Security Advancements

### 1. E2EE Invite Protections & Strict Expiry
To prevent passive link-hijacking and timing-based replay attacks:
*   **Time-Limited Invite Expiry:** Group creators can now generate links that enforce a strict **10-minute expiry validation**. Once expired, links are instantly blocked and rejected client-side.
*   **Optional E2EE Password/PIN Protection:** Admins can encrypt invite credentials with a password. Joining clients must enter this password, which is used to securely decrypt invite payloads on-device before starting the group join handshake.

### 2. Encrypted Local Group Audit Log
For compliance, auditing, and complete trust transparency without cloud leaks:
*   **Local Page-Encrypted Ledger:** Critical administrative actions—such as **member revocations** and **admin message deletions**—are chronological ledger entries stored in the device's SQLCipher database.
*   **Visual Timeline Render:** A premium visual security timeline cards UI component has been added to Group Settings, displaying decrypted audit trails directly on-device.

### 3. MLS-Aligned E2EE Media Attachments
Seamless secure file-sharing in group chats:
*   **Symmetric On-Device Encryption:** Media files (e.g. images) are encrypted symmetrically on the device before transmission.
*   **MLS E2EE Relay Envelopes:** Encrypted payloads are formatted inside secure JSON media envelopes and relayed within MLS-aligned E2EE sessions. The relay server remains amnesiac and state-free, completely unable to view or parse the media contents.

### 4. Codebase Stabilization & Strict Type-Safety
*   **100% Compiler Verification:** The entire TypeScript codebase compiles cleanly (`npx tsc --noEmit` returns exit code 0).
*   **Improved Error Mitigation:** Replaced dynamic roster-lookup edge cases with strict type annotations, eliminating possible runtime reference errors.

## 🛠️ Verification Checklist

Ensure the following security verification passes in your local build:
1.  **Time-Limited Invites:** Generate an invite link with the 10-minute limit active. Set your device's system clock forward and verify that joining is strictly blocked with the *"Invite Link Expired!"* warning.
2.  **Invite Password:** Protect an invite with a custom password. Verify that entering an incorrect PIN fails client-side, whereas entering the correct password seamlessly decrypts and executes the join.
3.  **Local Auditing:** Delete a message or revoke a member. Open Group Details and verify that a new audit row dynamically prints to the decrypted ledger card.
4.  **Media Sharing:** Send an image in an MLS group chat. Confirm that it encrypts on-device, successfully travels through the stateless relay, and decrypts/renders instantly for group recipients.
